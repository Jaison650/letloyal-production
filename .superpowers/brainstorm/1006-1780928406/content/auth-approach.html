How should customers log in?

Pick the authentication approach for the India pilot. Each has a different friction level.

A

📱 Phone + Password Recommended

Customer registers with phone + password. Login = phone + password. Forgot password sends a reset link to their email (if they've added one) or shows a "contact support" fallback.

✓ Familiar UX ✓ Secure ~ Requires email for reset
B

🔢 Phone + OTP (no password)

Customer enters phone → gets a 6-digit OTP via SMS → logged in. No password needed. OTP is the auth. Needs an SMS gateway (Twilio / MSG91 / Fast2SMS).

✓ No passwords to forget ✓ Very common in India ✗ Costs money per SMS ✗ Needs SMS integration now
C

🔓 Phone only (current — no password)

Keep the existing system: phone number is the only identity. Anyone who knows a phone number can view that customer's rewards. Simple but not secure for a multi-merchant setup.

✓ Zero friction ✗ No security ✗ Can't have a profile
